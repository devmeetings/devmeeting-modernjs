Problemem z var w JS jest zasięg funkcyjny. Czyli:

      console.log(k); // undefined (even in strict mode)

      for (var k in [0, 1, 2]) {
        ...     
      }

      console.log(k); // 2

ES2015 wprowadza nowe sposoby deklarowania zmiennych (i stałych).

Zarówno const jak i let mają zasięg blokowy.
